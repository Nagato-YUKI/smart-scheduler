## 系统简介

智能排课系统是一款基于 Web 的课程表管理系统，能够根据教室、教师、班级、课程等基础数据，自动生成科学合理的课程安排。

### 核心特性

- **智能排课算法**：自动分配教室和上课时间
- **节假日处理**：自动避开节假日安排课程
- **教师负载均衡**：确保每位教师工作量合理
- **教室利用率优化**：提高教室资源使用效率
- **集中排课**：同一教师的课程尽量安排在连续天数
- **数据导入导出**：支持 Excel 批量导入

## 安装部署

### 1. 克隆或下载项目

```bash
git clone <repository_url>
cd paike
```

### 2. 安装后端依赖

```bash
cd backend
pip install -r requirements.txt
```

### 3. 安装前端依赖

```bash
cd frontend
npm install
```

### 4. 初始化数据库

```bash
cd backend
python init_db.py
```

这将创建 SQLite 数据库并插入默认节假日数据。

## 快速开始

### 1. 启动后端服务

```bash
cd backend
python app.py
```

后端服务将在 `http://127.0.0.1:5000` 启动。

### 2. 启动前端服务（新窗口）

```bash
cd frontend
npm run dev
```

前端服务将在 `http://localhost:3000` 启动。

### 3. 访问系统

在浏览器中打开 `http://localhost:3000`

### 4. 导入测试数据

1. 点击左侧菜单「数据导入」
2. 下载综合模板
3. 填写数据后上传
4. 点击「智能排课」开始排课

## 功能模块详解

### 1. 首页仪表盘

展示系统概览数据：
- 教室总数
- 教师总数
- 班级总数
- 课程总数

### 2. 教室管理

管理教学场所：
- 添加/编辑/删除教室
- 设置教室容量
- 设置教室类型（普通教室、多媒体教室、机房、实验室）

### 3. 教师管理

管理授课教师：
- 添加/编辑/删除教师
- 设置可授课程
- 设置每周最大课次数

### 4. 班级管理

管理教学班：
- 添加/编辑/删除班级
- 设置学生人数
- 设置所属院系

### 5. 课程管理

管理课程信息：
- 添加/编辑/删除课程
- 设置课程类型（普通授课、上机、实验）
- 设置总课时

### 6. 节假日管理

管理学期内节假日：
- 添加/编辑/删除节假日
- 系统自动避开节假日排课
- 默认包含中秋节、国庆节、元旦、春节

### 7. 数据导入

批量导入数据：
- 支持单个类型导入（教室/教师/班级/课程/节假日）
- 支持综合导入（一次性导入所有数据）
- 提供 Excel 模板下载

### 8. 智能排课

核心排课功能：
- 自动分配教室和上课时间
- 考虑教室容量和类型
- 避开节假日
- 均衡教师工作负载
- 均衡每天课时分布
- 集中同一教师的课程

### 9. 课表查看

多维度查看课表：
- 按班级查看
- 按教师查看
- 按教室查看

### 10. 课时统计

查看排课统计数据：
- 总课时数
- 时段分布（上午/下午/晚上）
- 每天分布
- 每周分布
- 教师工作量统计
- 教室利用率统计

## 数据导入说明

### 综合导入模板

综合导入模板包含5个工作表：

1. **教室数据**：教室编号、名称、容量、类型
2. **教师数据**：工号、姓名、可授课程、每周最大课次数
3. **班级数据**：编号、名称、学生人数、院系
4. **节假日数据**：日期（YYYY-MM-DD格式）、名称、备注
5. **课程数据**：编号、名称、类型、课时、授课教师、授课班级

### 数据约束

| 数据项 | 约束说明 |
|--------|----------|
| 教室容量 | 必须 >= 班级学生人数 |
| 教师课程 | 必须与教师可授课程匹配 |
| 节假日 | 日期格式必须为 YYYY-MM-DD |
| 课程课时 | 影响排课周次（64课时=16周） |

### 导入失败处理

如果部分数据导入失败：
- 系统会显示成功导入数量
- 列出导入失败的行号和错误原因
- 已成功的数据不受影响

## 排课算法说明

### 排课策略

1. **教师优先**：带课多的教师先排
2. **集中排课**：同一教师的课程尽量安排在2-3天内
3. **日期均衡**：每天课时分布尽量均衡
4. **时段优先**：优先使用上午/下午时段，晚上时段作为补充
5. **节假日回避**：自动跳过节假日

### 时段设置

| 时段 | 课时数 | 说明 |
|------|--------|------|
| 上午 | 4课时 | 第1-4节课 |
| 下午 | 4课时 | 第5-8节课 |
| 晚上 | 3课时 | 第9-11节课 |

### 教师工作负载

- 每位教师最多分配3天有课
- 避免教师五天都有课
- 每位教师带3-5个教学班

### 教室分配

- 根据班级人数分配合适容量的教室
- 优先使用利用率较低的教室
- 同一课程在同一教室上课

## 常见问题

### Q1: 排课失败怎么办？

**A**: 检查以下可能原因：
- 教师可授课程与分配的课程不匹配
- 班级数量超过教室容量
- 课时数过高导致排课周次不足

### Q2: 为什么某些教师五天都有课？

**A**: 系统会尽量避免，但受限于：
- 该教师教授的班级数量较多
- 教室资源紧张
- 节假日占用了一些时段

### Q3: 如何重新排课？

**A**:
1. 点击「数据导入」页面
2. 点击「清空所有数据」
3. 重新导入数据
4. 再次执行排课

### Q4: 晚上为什么没有课？

**A**: 晚上时段作为补充，主要用于：
- 64课时以上的课程补足
- 白天排不下的课程
- 教室利用率还有空间时

### Q5: 导入数据格式错误？

**A**: 确保：
- 日期格式为 YYYY-MM-DD
- Excel 文件为 .xlsx 格式
- 表头与模板一致
- 没有合并单元格
