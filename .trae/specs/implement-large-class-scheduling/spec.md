# 智能排课系统（大课制版）规格文档

## Why
原有排课系统采用"2课时为一个排课单元"的模式，现需改为大课制：上午4节、下午4节、晚上3节的大时段模式，每次课占一个完整大时段，一门课每周只排1次。需要重新设计整个排课系统以支持大课制的需求。

## What Changes
- 新增完整的智能排课系统，包含数据实体定义、自动排课算法、课表展示与导出功能
- 支持大课制排课：每天3个大时段（上午4节、下午4节、晚上3节）
- 每个教学班每周只排1次课，固定在一个完整大时段内
- 实现硬约束和软约束的排课算法
- 提供班级课表、教师课表、教室课表的可视化展示
- 提供统计面板显示课时完成情况和缺课提示
- 支持手动调整和导出功能

## Impact
- 新增排课能力
- 新增数据管理：教室、教师、班级、课程、节假日
- 新增排课算法引擎
- 新增课表可视化界面

## ADDED Requirements

### Requirement: 数据实体管理
系统需提供以下数据实体的管理功能：

#### 1. 教室信息管理
- **教室编号**：唯一标识
- **教室名称**：如"博学楼101"
- **容量**：座位数
- **教室类型**：普通教室、多媒体教室、机房、实验室等
- **可用状态**：默认可排课

#### 2. 教师信息管理
- **教师编号**：唯一标识
- **姓名**
- **可授课程列表**：多选，最多选择2门不同课程
- **授课班级上限**：最多5个班级
- **每周课次上限**：5次（每次为一整个大课时段）

#### 3. 班级信息管理
- **班级编号**：唯一标识
- **班级名称**：如"计科2301"
- **学生人数**
- **所属专业/年级**

#### 4. 课程信息管理
- **课程编号**：唯一标识
- **课程名称**
- **课程类型**：普通授课、实验、上机等，决定所需教室类型
- **总课时**：固定64课时
- **授课教师**：必须在教师名单内，且该课程在教师可授范围内
- **授课班级**：不支持合班，一个课程+一个班级+一名教师=一个教学班

#### 5. 节假日表管理
- **日期列表**：仅记录周一至周五的放假日期

### Requirement: 学期与时间设置
- 学期总周数：16周
- 每周上课日：周一至周五（周六、日休息）
- 每天大时段划分：
  - **上午**：第1–4节（4课时）
  - **下午**：第5–8节（4课时）
  - **晚上**：第9–11节（3课时，且不可拆分）
- 排课单位：每个教学班每周**只安排1次课**，固定在一个大时段内

### Requirement: 自动排课硬约束
排课算法必须满足以下硬约束：
1. **避开节假日**：节假日表中的日期禁用所有大时段
2. **教室固定不变**：每个教学班在整个学期使用同一教室
3. **教室容量**：教室座位数≥班级学生人数
4. **教室类型匹配**：必须匹配课程类型要求
5. **教师课次上限**：每位教师每周大课总次数≤5
6. **教师时段无冲突**：同一教师不能在同一大时段被安排到两个教学班
7. **教室时段无冲突**：同一教室同一大时段只能有一个教学班使用
8. **同班一天最多一次**：每个教学班每天最多安排1次课
9. **时段完整性**：只允许选择上午、下午、晚上三种大时段

### Requirement: 排课优化软约束
按优先级排序：
1. **教师课程集中**：同一教师所有教学班集中在连续几天内
2. **班级课表均衡**：同一班级的不同课程安排在不同天
3. **容量合理匹配**：尽量将班级安排在容量刚好的教室
4. **优先保证64课时**：优先安排上午/下午，晚上作为备选

### Requirement: 课表展示与统计
- **班级课表**：以周次为行、星期为列的表格展示
- **教师课表**：列出教师每周的所有上课安排
- **教室课表**：教室使用占用表
- **统计面板**：显示实际排入大时段明细、总课时、缺课课时提示

### Requirement: 手动调整与导出
- 支持拖拽更改教学班的周次或大时段
- 调整时实时校验冲突
- 导出Excel或直接打印课表

## Impact Analysis
- 这是一个全新的功能模块
- 需要前端界面、后端算法、数据存储三层架构
- 涉及排课算法的复杂实现
